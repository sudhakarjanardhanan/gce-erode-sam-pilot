# SAM Student Progress Report - Requirements Summary

Document status: Agreed in design sessions, March 2026  
Scope: Pilot deployment, GCE Erode, Cycle 1

## 1. Purpose and Design Philosophy
The report is a contribution narrative, not a scorecard.

Central question:
"How did this student contribute to the work this semester - and what do they bring to it next cycle?"

Design principles:
- Psychological safety
- Growth-mindset language
- Specificity
- 2+1 framing (two strengths and one growth area)
- Contribution over performance

## 2. Structural Model
- Cycle: one semester
- Session: one 30-minute event with Presenter, Technical Reviewer, Feedback Strategist
- Student participates once per role per cycle (3 sessions total)

Implications:
- No trend within one cycle
- Cross-cycle comparison valid from Cycle 2 onward
- Roles are not comparable and must not be aggregated

## 3. Report Structure
- Page 1: Identity and cycle overview
- Page 2: Contribution as Presenter
- Page 3: Contribution as Technical Reviewer
- Page 4: Contribution as Feedback Strategist
- Page 5: Contribution summary narratives (one per role)
- Page 6: What to bring next cycle (one prompt per role)
- Final footer: fixed assessment statement

## 4. Journey Stage System (Per Role)
- <25%: Finding Your Ground
- 25-49%: Building Momentum
- 50-74%: Gaining Confidence
- >=75%: Leading the Room

Rules:
- One stage per role section
- No overall stage across roles
- No aggregate score across roles

## 5. AI Behavior and Faculty Override
- AI selects top 2 strengths per role
- AI selects 1 growth area per role
- AI drafts role narrative and next-cycle prompts
- Faculty can override strengths, growth area, and narratives before publishing
- Overrides are logged for future tuning

## 6. Explicit Exclusions (Pilot)
- Overall percentage across all roles
- Combined cross-role radar chart
- Raw point totals as primary student display
- Intra-cycle trend chart
- Anna University grade labels in student report
- Tamil language content (pilot deferred)

## 7. Format and Access
- Format: multi-page PDF
- Current access: faculty-triggered via Student Report flows
- Student login access: planned
- Language: English only
